ROBOSHOP E-commerce Microservices

Project Overview

  • Roboshop is a fully functional ecommerce platform built with microservices using a 3-tier scalable architecture. Users can browse products, manage carts, view orders, and ship globally.
  • It uses NGINX as a reverse proxy with microservices built in Node.js, Java, and Python, supported by databases MongoDB, Redis, RabbitMQ, and MySQL.
  • Deployment is automated using Shell scripting, Ansible, Terraform, Docker, and Jenkins, with monitoring through Grafana, Prometheus, cAdvisor, and Node Exporter.

Architecture & Tools

  • Architecture: Web (NGINX), Application (microservices), and Database tiers isolated in custom VPC subnets with secure load balancing and auto-scaling.
  • Infrastructure: Provisioned using Terraform with spot instances and NAT instance for cost efficiency.
  • CI/CD: Automated Jenkins pipelines to build and deploy Docker containers triggered by GitHub commits.
  • Configuration: Ansible roles and shell scripts automate service setup and deployment.
  • Monitoring: Load balancer access logs are sent to CloudWatch Logs. Additionally, a basic monitoring stack is set up using Grafana, Prometheus, cAdvisor, and Node Exporter to track container and host-level metrics.
